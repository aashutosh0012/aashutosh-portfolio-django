Jira-Solarwinds-Integration-via-REST-API

Source Code on Github: https://github.com/aashutosh0012/Jira-Solarwinds-Integration-via-REST-API .

Summary

SolarWinds Application collects all Alerts about any Objects in Different Tables in SolarWinds Database. The Alerts are stored in some tables only when the Alerts are Active, when Alert is resolved, Alerts is moved out of Active Alerts Table and recorded in Historical Tables. In Order to integrate SolarWinds Monitoring in Jira Service Desk for Ticket Creation for each alert, we:

  • Read Active Alerts from SolarWinds Database.
  • Store All Alerts in an external Database (INTERMEDIATE DB).
  • Compare if the New Alerts is already present in INTERMEDIATE DB.
  • If the Alert is New, Insert Alert Data in a INTERMEDIATE DB Table which store all information along with Alert Current Status, and create a Ticket in Jira SD via REST API.
  • If the Alert is already present, Ignore the Alert, as the Alert is still active and Ticket has been created already in Jira Service Desk.
  • If the Alert is not present in Active Alerts, but present in INTERMEDIATE DB Table, it means, Alert has resolved ,then Resolve the Ticket in Jira SD.
